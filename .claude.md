# Project Guidelines for Claude

## Component Development

### Always Check for Existing Components First

**CRITICAL**: Before creating any new component, button, input, or UI element:

1. **Search the codebase** for existing components using Grep or Glob
2. **Check `/src/components/`** directory for reusable components
3. **Review the component library** to understand available variants and props
4. **Reuse existing components** whenever possible by:
   - Using different variants (e.g., `variant="outlined"` vs `variant="primary"`)
   - Passing different props (e.g., `icon`, `size`, etc.)
   - Combining existing components rather than duplicating code

### Component Checklist

Before creating a new component, verify:
- [ ] No existing component serves this purpose
- [ ] Existing components cannot be extended with props/variants
- [ ] The new component will be reused in multiple places

### Examples of Existing Components

- **Button** - Multiple variants: `standard`, `primary`, `ghost`, `outlined`, `text`, `ai`
- **Icon** - 200+ icons from FontAwesome + Lucide
- **Dropdown** - Custom styled dropdown
- **TextInput** - Various input types
- **Avatar** - Multiple sizes
- **EmployeeCard**, **Gridlet**, etc.

## Styling Guidelines

- Use **CSS variables** from `/src/index.css` for colors, spacing, shadows
- Follow **existing patterns** for dark mode support
- Use **Tailwind classes** combined with inline styles for design tokens
- Maintain **consistency** with existing component styles

## Icon Usage

- Check `Icon.tsx` for available icons before adding new ones
- Use existing icon names when possible
- For new icons:
  - Add to FontAwesome imports if available in free package
  - Use Lucide for outlined/lined icons
  - Update IconName type and iconMap

## Best Practices

1. **Read before writing** - Always read existing files to understand patterns
2. **Reuse over recreate** - Leverage existing components and utilities
3. **Consistency** - Match existing code style and conventions
4. **CSS Variables** - Use design tokens from index.css
5. **Dark mode** - Ensure all components work in both themes
